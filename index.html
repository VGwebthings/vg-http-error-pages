Things will go wrong. We at • fortrabbit like meaningful error messages. Here are our custom error pages for our PHP PaaS. They are all stand alone pages – CSS is included in the head. The syntax is hopefully valid HTML5. Feel free to rip, mix & burn.

Custom HTTP Error Pages

The most common HTTP Status Apache error pages with some additional infos for humans:

  • 401 – Unauthorized, Login failed
  • 403 – Forbidden
  • 404 – File not Found
  • 410 – Gone
  • 500 – Internal Server Error
  • 503 – Service Unavailable / Overload
  • 505 – HTTP Version Not Supported

Other Error and Success Pages

We have a few other custom pages that are not handeld by the standard HTTP Errors:

  • Service Offline - When we have shut down an app - probably for security reasons
  • Missing Routing - When an internal or external domain is not yet or not correctly configured
  • Service Unavailable – Set Up, Maintenance, or when some somehting when wrong somewhere
  • App Frozen – when a developer app is currently not available
